Project Template - ASP.NET Core Application Razor Pages with LHQ

VS2017 VS2019

  • This project template is compatible with Visual Studio 2017 and Visual Studio 2019
  • Project uses latest csproj project format used in NET Core apps (2.x and above)
  • Localization HQ Editor is available also for Visual Studio 2015 if you have NET Core 1.x app

Project template to create ASP.NET Core Web Application Razor Pages with LHQ editor files and registrations in Startup class.

Visual Editor

  • File Strings.lhq (project root) is container for all string resources in application
  • File extension (lhq) is handled by LHQ extension loaded into Visual Studio Editor
  • Visual Studio shows editor pane where you can visually edit all string resources for all languages your application requires

Code Generator

  • File Strings.lhq.tt is responsible for generating NET resource files (*.resx) and strongly typed C# files (*.cs)
  • Each time file Strings.lhq is saved in LHQ editor, code generation is triggered automatically by executing file Strings.lhq.tt (T4 Text Templating)
  • File Strings.lhq.tt can be edited to configure generator settings, like destination folder where to produce generated files, format of output, etc

Integration with ASP.NET Core application

To utilize the most from LHQ editor extension, an Nuget package ScaleHQ.AspNetCore.LHQ was created.
This nuget package provides extensions to ASP.NET Core system. Extensions are made to IServiceCollection , IApplicationBuilder and others to do registration of StringsModel.

Nuget package ScaleHQ.AspNetCore.LHQ is already installed in project template.
Building project will restore this package.
In some cases manual restore of nuget packages is required (running command dotnet restore).


Sample project description

File Startup.cs :

  • services.AddTypedStringsLocalizer<StringsModelLocalizer, StringsModel>();
    Register generated strings model and localizator into ASP.NET Core services.
  • services.AddMvc().AddMvcTypedStringsLocalizer()
    Configure MVC related stuff (like data annotations, etc) to work with generated strings model and localizer.
  • app.UseTypedStringsLocalizer<StringsModelLocalizer, StringsModel>();
    Tell ASP.NET Core services to use previously registered typed strings model and localizer.

File Pages\Index.cshtml :

  • @StringsModel.Messages.PleaseEnterInfo
    Returns localized text according to current culture using strongly type class StringsModel.
  • @Model.GetWelcomeMessage()
    Returns localized text according to current culture but actual value is bind-ed in view model file HomeViewModel.cs
  • <label asp-for="Username"...
    Such label is bind-ed to property Username in view model HomeViewModel.
    This property has data annotations configured to return localized texts (from StringsModel):
    • [DisplayAttribute] returns localized text for display name of property
    • [RequiredAttribute] returns localized text for custom error message when field value is not specified
    • [OnlyAlphanumericAttribute] returns localized text with error message when field value is not alpha numeric value.
