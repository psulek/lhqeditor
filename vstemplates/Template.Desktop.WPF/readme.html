Visual Studio Extension / Project Template

Project Template - Windows WPF Application with LHQ

VS2015 VS2017 VS2019

  • This project template is compatible with Visual Studio 2015, 2017 and 2019
  • Project template showcase usage of LHQ Editor in Windows Desktop WPF application

Visual Editor

  • File Strings.lhq (project root) is container for all string resources in application
  • File extension (lhq) is handled by LHQ extension loaded into Visual Studio Editor
  • Visual Studio shows editor pane where you can visually edit all string resources for all languages your application requires

Code Generator

  • File Strings.lhq.tt is responsible for generating NET resource files (*.resx) and strongly typed C# files (*.cs)
  • Each time file Strings.lhq is saved in LHQ editor, code generation is triggered automatically by executing file Strings.lhq.tt (T4 Text Templating)
  • File Strings.lhq.tt can be edited to configure generator settings, like destination folder where to produce generated files, format of output, etc

Integration with Windows Desktop WPF application

To utilize the most from LHQ editor extension, an Nuget package ScaleHQ.WPF.LHQ was created.
This nuget package provides custom extension to WPF markup to be used in XAML.
This sample project contains usage of this markup extension.

Nuget package ScaleHQ.WPF.LHQ is already installed in project template.
Building project will restore this package.
In some cases manual restore of nuget packages is required (uninstall and then install back).


Sample project description

File App.xaml.cs :

  • LocalizationExtension.Initialize(...);
    Registers generated class StringsContext into WPF system to be able to use localization markup extension.
  • StringsContext.Instance.Culture = new CultureInfo("en");
    Sets current culture to English on StringsContext class.

File MainWindow.xaml :

  • <TextBlock Text="{lhq:Localization Key={x:Static resources:StringsKeys+Messages.Welcome}}"/>
    Renders localized text user specified resource key, according to current culture.
  • <ItemsControl ItemsSource="{Binding Path=AvailableCultures}">
    Populates list of radio buttons with localized text binded in model using StringsContext class.
  • <lhq:Localization Key="{x:Static resources:StringsKeys+Messages.Title}">
        <Binding Path="WindowsUser" />
        <Binding Path="." Source="{StaticResource StaticString}" />
        <Binding Path="Now" Source="{x:Static system:DateTime.Now}"
                 StringFormat="{}{0:dd/MM/yyyy (dddd)}" />
    </lhq:Localization>

    Renders localized text with C# string parameters dynamically binded values using XAML <Binding .../> expression.
